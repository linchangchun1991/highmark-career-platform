1. 访问：https://supabase.com/dashboard
2. 登录你的账号

### 1.2 创建数据表

1. 点击左侧菜单 **"SQL Editor"**
2. 点击 **"New query"**
3. **复制粘贴**以下代码：

```sql
CREATE TABLE IF NOT EXISTS jobs (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  raw_text TEXT NOT NULL,
  company TEXT NOT NULL,
  roles TEXT NOT NULL,
  location TEXT NOT NULL,
  link TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_jobs_created_at ON jobs(created_at DESC);

ALTER TABLE jobs ENABLE ROW LEVEL SECURITY;

DROP POLICY IF EXISTS "Allow public read access" ON jobs;
CREATE POLICY "Allow public read access" ON jobs
  FOR SELECT USING (true);

DROP POLICY IF EXISTS "Allow public insert" ON jobs;
CREATE POLICY "Allow public insert" ON jobs
  FOR INSERT WITH CHECK (true);
```

4. 点击右上角 **"Run"** 按钮（或按 `Ctrl+Enter`）
5. 看到 **"Success"** 表示成功 ✅

## ❌ 常见问题解决

### 问题 1：Vercel 部署失败，显示 "Build Error"

**解决方法**：
1. 检查环境变量是否正确添加
2. 确保所有变量名都是大写
3. 检查 Supabase URL 和 Key 是否正确复制（不要有多余空格）

### 问题 2：部署成功但网站显示错误

**解决方法**：
1. 确认 Supabase 数据库表已创建（参考第一步）
2. 检查环境变量是否在 Vercel/Zeabur 中正确配置
3. 查看部署日志：在 Vercel Dashboard → 你的项目 → "Deployments" → 点击最新的部署 → 查看 "Build Logs"

### 问题 3：数据库连接失败

**解决方法**：
1. 确认 Supabase 项目 URL 正确：`https://ysscurevsdenbmsnixpc.supabase.co`
2. 确认 API Key 正确（完整复制，不要遗漏）
3. 在 Supabase Dashboard 检查项目是否正常运行

### 问题 4：GitHub 推送失败

**解决方法**：
```bash
# 如果提示需要认证，执行：
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"

# 然后重新推送
git push -u origin main
```
