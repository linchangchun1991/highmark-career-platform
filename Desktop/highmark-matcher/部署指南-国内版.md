## 第一步：配置 Supabase 数据库（免费）

### 1.1 注册 Supabase 账号

1. 访问：https://supabase.com
2. 点击右上角 **"Start your project"** 或 **"Sign up"**
3. 使用 GitHub 账号登录（推荐）或邮箱注册
4. 完成邮箱验证

### 1.2 创建新项目

1. 登录后，点击 **"New Project"**
2. 填写项目信息：
   - **Name**: `highmark-matcher`（随便起名）
   - **Database Password**: 设置一个强密码（**重要：记住这个密码！**）
   - **Region**: 选择 **"Southeast Asia (Singapore)"**（离中国最近）
   - **Pricing Plan**: 选择 **Free**（免费版）
3. 点击 **"Create new project"**
4. 等待 2-3 分钟，项目创建完成

### 1.3 获取 API 密钥

1. 项目创建完成后，点击左侧菜单 **"Settings"**（齿轮图标）
2. 点击 **"API"**
3. 找到以下信息并**复制保存**：
   - **Project URL**: 类似 `https://xxxxx.supabase.co`
   - **anon public key**: 一长串字符，以 `eyJ...` 开头

### 1.4 创建数据库表

1. 在 Supabase Dashboard 左侧菜单，点击 **"SQL Editor"**
2. 点击 **"New query"**
3. 复制以下 SQL 代码并粘贴：

```sql
-- 创建 jobs 表
CREATE TABLE jobs (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  raw_text TEXT NOT NULL,
  company TEXT NOT NULL,
  roles TEXT NOT NULL,
  location TEXT NOT NULL,
  link TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 创建索引以优化查询性能
CREATE INDEX idx_jobs_created_at ON jobs(created_at DESC);

-- 允许公开访问（根据你的安全需求调整）
ALTER TABLE jobs ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow public read access" ON jobs
  FOR SELECT USING (true);

CREATE POLICY "Allow public insert" ON jobs
  FOR INSERT WITH CHECK (true);
```

4. 点击右上角 **"Run"** 或按 `Ctrl+Enter`
5. 看到 "Success. No rows returned" 表示成功 ✅

## 第四步：使用 Vercel 部署（推荐新手）

### 4.1 准备代码

1. 在项目根目录创建 `.env.local` 文件（如果还没有）
2. 打开 `.env.local`，填入你的配置：

```env
# Supabase 配置（从第一步获取）
NEXT_PUBLIC_SUPABASE_URL=https://你的项目ID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的anon key

# DeepSeek API 配置（从第二步获取）
DEEPSEEK_API_KEY=sk-你的deepseek key
```

3. **重要**：`.env.local` 文件不要提交到 Git（已经在 .gitignore 中）

### 4.2 将代码推送到 GitHub

1. 如果没有 GitHub 账号，先注册：https://github.com
2. 在 GitHub 创建一个新仓库（Repository）
3. 在项目目录执行：

```bash
cd /Users/changchun/Desktop/highmark-matcher

# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 添加远程仓库（替换成你的仓库地址）
git remote add origin https://github.com/你的用户名/highmark-matcher.git

# 推送代码
git push -u origin main
```

### 4.3 在 Vercel 部署

1. 访问：https://vercel.com
2. 使用 GitHub 账号登录
3. 点击 **"Add New Project"**
4. 选择你刚才创建的 GitHub 仓库
5. 配置项目：
   - **Framework Preset**: Next.js（自动检测）
   - **Root Directory**: `./`（默认）
6. **重要：添加环境变量**
   - 在 **"Environment Variables"** 部分
   - 点击 **"Add"**，添加以下变量：
     - `NEXT_PUBLIC_SUPABASE_URL` = 你的 Supabase URL
     - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = 你的 Supabase Key
     - `DEEPSEEK_API_KEY` = 你的 DeepSeek Key
7. 点击 **"Deploy"**
8. 等待 2-3 分钟，部署完成 ✅
9. 你会得到一个网址，类似：`https://highmark-matcher.vercel.app`

### 4.4 测试部署

1. 访问你得到的网址
2. 测试登录功能
3. 测试 BD 端添加岗位
4. 测试教练端匹配功能

## 第五步：使用阿里云部署（国内访问快）

### 5.1 购买服务器

1. 访问：https://www.aliyun.com
2. 注册/登录账号
3. 完成实名认证
4. 进入 **"云服务器 ECS"**
5. 点击 **"立即购买"**
6. 选择配置：
   - **地域**：选择离你最近的（如：华东1）
   - **实例规格**：选择 **"共享型"** → **"ecs.t6-c1m2.large"**（2核2G，约 50-80元/年）
   - **镜像**：选择 **"Ubuntu 22.04"**
   - **系统盘**：40GB
7. 设置 root 密码（记住！）
8. 完成购买

### 5.2 连接服务器

1. 在 ECS 控制台，找到你的服务器
2. 复制 **公网 IP**
3. 使用 SSH 连接（Mac/Linux 用终端，Windows 用 PuTTY）：

```bash
ssh root@你的公网IP
```

4. 输入 root 密码

### 5.3 安装 Node.js 和依赖

在服务器上执行：

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt-get install -y nodejs

# 验证安装
node -v  # 应该显示 v18.x.x
npm -v

# 安装 PM2（进程管理器）
npm install -g pm2

# 安装 Git
apt install git -y
```

### 5.4 部署代码

```bash
# 克隆你的代码（替换成你的 GitHub 仓库地址）
git clone https://github.com/你的用户名/highmark-matcher.git

# 进入项目目录
cd highmark-matcher

# 安装依赖
npm install

# 创建环境变量文件
nano .env.local
```

在编辑器中粘贴：

```env
NEXT_PUBLIC_SUPABASE_URL=https://你的项目ID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的anon key
DEEPSEEK_API_KEY=sk-你的deepseek key
```

按 `Ctrl+X`，然后 `Y`，然后 `Enter` 保存。

### 5.5 构建和启动

```bash
# 构建项目
npm run build

# 使用 PM2 启动（后台运行）
pm2 start npm --name "highmark-matcher" -- start

# 设置开机自启
pm2 startup
pm2 save
```

### 5.6 配置防火墙和安全组

1. 在阿里云控制台，找到你的服务器
2. 点击 **"安全组"** → **"配置规则"**
3. 添加规则：
   - **端口范围**：3000
   - **授权对象**：0.0.0.0/0
   - **协议类型**：TCP
4. 保存

### 5.7 访问系统

访问：`http://你的公网IP:3000`

## 第六步：配置域名（可选，但推荐）

### 6.1 购买域名

1. 在阿里云或腾讯云购买域名（约 10-50 元/年）
2. 完成域名实名认证

### 6.2 解析域名

1. 在域名管理控制台，添加 **A 记录**：
   - **主机记录**：`@` 或 `www`
   - **记录值**：你的服务器公网 IP
   - **TTL**：600

### 6.3 配置 Nginx（反向代理）

在服务器上安装 Nginx：

```bash
apt install nginx -y

# 创建配置文件
nano /etc/nginx/sites-available/highmark-matcher
```

粘贴以下配置（替换 `yourdomain.com` 为你的域名）：

```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

保存后：

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/highmark-matcher /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重启 Nginx
systemctl restart nginx
```

### 6.4 配置 HTTPS（免费 SSL 证书）

```bash
# 安装 Certbot
apt install certbot python3-certbot-nginx -y

# 申请证书
certbot --nginx -d yourdomain.com -d www.yourdomain.com

# 按照提示完成配置
```

完成后，访问：`https://yourdomain.com`
